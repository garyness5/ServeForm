# Quotation Workflow

## Purpose

Quotation is the Sales service built on top of Events.

It takes the current operational Event and lets Sales create one or more customer-facing commercial proposals without changing Kitchen workflow.

Quotation is not:
- CRM
- Accounting
- Tax calculation
- Messaging
- Approval workflow
- Sales pipeline
- Production management
- Inventory
- Purchase orders

Savveyra provides the operational truth. Companies continue to communicate through their normal channels such as phone, email, meetings, Teams, Slack, or their own CRM.

## Core Philosophy

Kitchen owns the Event.

Sales owns the Quote.

The Event is the operational workspace. It contains the menus, guests, customer, venue, date, format, and production-facing information.

The Quote is a commercial document created from the Event. It contains selling prices, quote notes, service charges, deposits, terms, and customer-facing output.

Kitchen should not manage quote revisions. Sales should not manually rebuild operational data. Supabase bridges the two.

## QuotationList

QuotationList is an Event-centered Sales inbox.

It is not a list of individual quotes.

Each row represents one Event in the quotation flow.

Example:

```text
Event       Customer      Status      Drafts   Issued   Accepted   Kitchen
BMO1        BMO           Accepted    2        1        SAV-001002 Current
HAC01       BMO           Sent        1        0        -          Update
```

Opening a row opens the Quotation workspace for that Event.

## Quote Workspace

The Quotation page is a workspace for one Event.

Inside the workspace, Sales can see:
- Event/customer/venue/date information
- Kitchen Update status
- all quotes for that Event
- the currently selected quote
- menus and menu contents
- financial summary
- deposits
- notes
- terms
- print/issue/accept actions

One Event can have many quotes:
- Draft
- Draft
- Issued SAV-001001
- Accepted SAV-001002

The Event is the folder. Quotes are documents inside that folder.

## Kitchen Update

Kitchen Update is the user-facing wording for Event revision changes.

Internally, Supabase tracks:
- `evt_items.quotation_revision_no`
- `quo_events.imported_event_revision_no`

If the Event revision is newer than the quote’s imported revision, Sales sees:

```text
Kitchen Update Available
```

This only means the Event changed since the quote was last imported.

It does not mean:
- automatic overwrite
- automatic new quote
- automatic approval
- internal message
- workflow task

Sales decides what to do.

## Kitchen Update Actions

When a Draft quote has a Kitchen Update available, Sales chooses:

### Import into Current Draft

The selected Draft quote is refreshed from the latest Event.

Manual values are preserved when possible:
- price per guest
- menu notes

This is used when Sales wants to continue working on the same customer proposal.

### Import as New Draft

A new Draft quote is created for the same Event using the latest Event state.

This is used when Sales wants to keep the current quote and create another proposal.

### Cancel / Ignore

Nothing changes.

Sales can come back later.

## Update All / Import Rule

Update All applies only to the open/current Draft quote.

Draft quotes may change every time they are updated/imported from the Event.

Issued and Accepted quotes never update from the Event.

If the user wants to change an Issued or Accepted quote:
1. Duplicate it.
2. Work on the new Draft.
3. Issue.
4. Accept if appropriate.

## Quote Status Lifecycle

Quote statuses are:

```text
Draft
Issued
Accepted
```

### Draft

Editable Sales working document.

Sales can:
- update from Event
- change selling prices
- change pricing mode
- edit quote reference
- edit service charge
- edit deposit amounts and due dates
- edit customer notes
- edit internal notes
- edit terms
- save repeatedly
- close and reopen
- duplicate
- issue

Draft quotes are not official customer documents.

### Issued

Locked customer-facing quote.

On Issue:
- Supabase generates official quote number.
- Quote is locked.
- `issued_at` is stamped.
- current imported costs, guests, menus, terms, prices, notes, deposits, allergens, and diet tags remain frozen as the historical issued quote.

Issued quotes can be printed repeatedly.

Issued quotes cannot be edited.

### Accepted

Locked quote approved by customer.

Accepted means:
- this is the customer-approved commercial proposal
- this is the current accepted quote for the Event
- the Event status becomes Accepted

Only one quote per Event should be the current Accepted quote. If another quote is accepted, the previous Accepted quote returns to Issued and remains historical.

Accepted quotes cannot be edited.

## Quote Numbering

Quote numbers are generated by Supabase, not Appsmith.

Quote number remains blank while quote is Draft.

On Issue:
- Supabase calls `get_next_document_number(client_id, 'quote')`
- quote number is assigned
- document sequence is incremented
- quote is locked

Example:

```text
SAV-001001
SAV-001002
SAV-001003
```

Quote numbers are unique per client and are never reused.

Quote Reference is separate and manual. It can be used for business/customer context such as:
- BMO-HAC01
- Smith Wedding
- RBC Tender

## Pricing Model

Quotation supports two pricing modes.

### Per Menu

Each menu has:
- guest count
- price per guest
- calculated line total

Formula:

```text
Guests × Price Per Guest = Line Total
```

Subtotal is the sum of active menu line totals.

### Event Total

Menus still display as the customer-facing breakdown, but Sales enters one total Event price.

## Financial Model

Quotation is visual/commercial tracking only. It is not accounting.

There is no tax calculation.

Financial fields:
- Subtotal
- Service Charge
- Quote Total
- Deposit 1
- Deposit 2
- Deposit 3
- Balance
- Final Due Date

Formula:

```text
Quote Total = Subtotal + Service Charge
Balance = Quote Total - Deposits
```

Deposits are manual. Balance is calculated.

Taxes, if applicable, are handled outside Savveyra.

## Menus in Quotation

Quotation displays each Menu as a customer-facing block.

Example:

```text
Menu Name
Diet Tags
Guests
Price / Total

Included:
  Dish
  Recipe
  Ingredient
  Ingredient

Allergens:
  Milk, Celery, Mustard
```

Menu contents come from the Menu. They are not manually recreated in Quotation.

## Allergens and Diet Tags

Diet Tags belong to the Menu as a whole.

A Vegan ingredient inside a Menu does not make the Menu Vegan.

A Menu is Vegan, Kosher, Gluten Free, etc. only if the Menu itself is assigned that Diet Tag.

Allergens are unique rollups for the Menu.

If a Menu is supposed to be dairy-free, Dairy should not appear in the allergen list.

Quotation displays:
- Menu-level Diet Tags
- unique Menu Allergens

## Relationship to Events

Events feed Quotation.

Event statuses relevant to Quotation:
- Sent
- Ordered
- Accepted

Quotation does not delete or deactivate quotes when Event status changes.

Closed Event behavior will be reviewed later as part of the final Event/archive workflow.

## Relationship to Groceries

Events branch downstream into:
- Groceries for purchasing/operations
- Quotation for Sales/customer proposal

Quotation and Groceries are separate services built on Events.

Quotation does not replace Groceries.

Groceries does not rely on quote pricing.

## Scope Boundary

Savveyra communicates operational changes through data, not through messaging.

Savveyra informs users that changes exist.

Savveyra does not manage conversations between departments.

Companies continue using their existing communication methods.

Savveyra remains the operational source of truth.

## Supabase-Owned Business Rules

Business rules are enforced in Supabase, not Appsmith.

Supabase owns:
- quote lifecycle
- quote locking
- quote numbering
- accepted quote behavior
- imported Event revision
- Kitchen Update detection
- financial calculations
- duplicate quote behavior
- update/import behavior
- edit protection for Issued/Accepted quotes

Appsmith owns:
- displaying data
- buttons
- modals
- warnings
- navigation
- user selection
